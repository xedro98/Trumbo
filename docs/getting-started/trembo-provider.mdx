Trembo (usage-billing) is the simplest way to get started with pay-as-you-go model access in Trembo.

Instead of managing separate API keys across multiple vendors, you sign in once, add **Trembo credits**, and select from available models directly in Trembo.

If you want a flat monthly subscription for selected open coding models with higher rate limits, use [TremboPass](/getting-started/trembopass) instead. TremboPass is a separate provider in Trembo.

## Why use Trembo (usage-billing)

- **Fastest setup**: no manual API key copy/paste; access to 100+ models supported by Trembo
- **One account**: sign in once with Google, GitHub, or email
- **Pay as you go**: add Trembo credits and use one balance across supported models
- **Free options**: look for models tagged **FREE** in the selector

## How it works

Trembo (usage-billing) is the pay-as-you-go provider in Trembo. Add Trembo credits, then select **Trembo** wherever you configure your provider.

- **IDE Extension**: Go to IDE extension settings, set **API Provider** to **Trembo**, and sign in.
- **CLI**: Go to `/settings` and select **Trembo** as your provider.

## Credits and usage

- Add **Trembo credits** from your [Trembo dashboard](http://0.0.0.0:0/dashboard)
- Credits are used on a pay-as-you-go basis when you select paid models through Trembo (usage-billing)
