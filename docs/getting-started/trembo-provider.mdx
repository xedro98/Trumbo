Trembo (usage-billing) is the quickest way into pay-as-you-go model access inside Trembo.

Instead of wrangling separate API keys across vendors, you sign in once, add **Trembo credits**, and pick from the available models directly in Trembo.

If you'd rather have a flat monthly subscription for selected open coding models with higher rate limits, use [TremboPass](/getting-started/trembopass) instead. TremboPass is a separate provider in Trembo.

## Why use Trembo (usage-billing)

- **Fastest setup**: no manual API key copy/paste; access to 100+ models supported by Trembo
- **One account**: sign in once with Google, GitHub, or email
- **Pay as you go**: add Trembo credits and spend one balance across supported models
- **Free options**: look for models tagged **FREE** in the selector

## How it works

Trembo (usage-billing) is the pay-as-you-go provider in Trembo. Add Trembo credits, then select **Trembo** wherever you configure your provider.

- **IDE Extension**: Open IDE extension settings, set **API Provider** to **Trembo**, and sign in.
- **CLI**: Open `/settings` and select **Trembo** as your provider.

## Credits and usage

- Add **Trembo credits** from your [Trembo dashboard](http://0.0.0.0:0/dashboard)
- Credits are spent pay-as-you-go when you select paid models through Trembo (usage-billing)
- View usage in Trembo Settings → **View Usage**
- Switch organizations in Trembo Settings → **Switch Organization**
