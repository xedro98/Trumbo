## Prerequisites

- Trembo CLI installed (install via `npm i -g trembo`)
- A provider authenticated (`trembo auth`) — use the Trembo Provider, TremboPass, or your own provider key ([Authorization Guide](/getting-started/authorizing-with-trembo#cli-setup))

## Quick Start

```bash
# Interactive session
trembo

# Run one task immediately
trembo "refactor this module to use async/await"

# Structured output for scripts
trembo --json "list TODO comments"
```

## Headless mode

Use headless mode for scripts and automation where you want processable output.

Headless kicks in when you pass flags like `--json`, when stdin is piped, or when output is redirected.

#### When headless activates

| Invocation | Reason |
|---|---|
| `trembo --json "task"` | JSON output mode |
| `cat file \| trembo "task"` | stdin is piped |
| `trembo "task" > output.txt` | stdout is redirected |

```bash
# CI/script style execution
git diff | trembo "review these changes"

# JSON for parsing
trembo --json "summarize this changelog" | jq -r '.text'
```

See: [Headless Mode](#headless-mode)

### Autonomous execution

For fully unattended runs, turn on auto-approval:

```bash
trembo --auto-approve true "run tests and fix failures"
```

Mode selection works in headless runs too:

```bash
# plan-first
trembo -p "design migration plan"

# act immediately (default)
trembo "apply migration"
```

<Warning>
Autonomous execution can modify files and run commands without further prompts. Run it on a clean branch and review the results.
</Warning>

## High-Value Commands

```bash
trembo --help
trembo <command> --help
```

| Command | Purpose |
|---|---|
| `trembo` | Start interactive mode or run a prompt |
| `trembo auth` | Authenticate and set provider/model |
| `trembo config` | Open the interactive config view |
| `trembo mcp` | Manage MCP servers |
| `trembo doctor` | Diagnose/fix configuration issues |
| `trembo history` | Show and manage task history |
| `trembo schedule` | Manage scheduled tasks |
| `trembo hub` | Manage local hub daemon |
| `trembo kanban` | Launch Kanban app |

## Most-Used Global Flags

Source of truth: [CLI Reference](/cli/cli-reference)

| Flag | Purpose |
|---|---|
| `-p, --plan` | Start in Plan mode |
| `--auto-approve <boolean>` | Global tool auto-approval (`true`/`false`, default `true`) |
| `-m, --model <model>` | Override model for this run |
| `-P, --provider <id>` | Override provider for this run |
| `-c, --cwd <path>` | Set working directory |
| `--config <dir>` | Use config directory |
| `--data-dir <dir>` | Use isolated local state |
| `--json` | Output newline-delimited JSON messages |
| `--thinking <level>` | Set reasoning effort: `none\|low\|medium\|high\|xhigh` (default `medium`) |
| `-t, --timeout <seconds>` | Set task timeout |

## Automation Patterns

### Pipe context in

```bash
cat README.md | trembo "summarize key setup steps"
git diff | trembo "review for potential regressions"
```

### Chain tasks

```bash
git diff | trembo "explain these changes" | trembo "write a commit message"
```

### Restrict command execution

```bash
export TREMBO_COMMAND_PERMISSIONS='{"allow": ["npm *", "git *"], "deny": ["rm -rf *", "sudo *"]}'
```

### Include images in tasks

```bash
trembo -i "fix the layout issue shown in @./screenshot.png"

# or reference inline
trembo "fix the UI shown in @./design-mockup.png"
```

### Set execution timeout

```bash
trembo --timeout 600 "run full test suite"
```

## JSON Output Schema

With `--json`, each line is a JSON message object.

```json
{"type":"say","text":"I'll create the file now.","ts":1760501486669,"say":"text"}
```

| Field | Type | Description |
|---|---|---|
| `type` | `"ask"` or `"say"` | Message category |
| `text` | `string` | Message content |
| `ts` | `number` | Unix timestamp in milliseconds |
| `say` | `string` | Subtype when `type` is `"say"` |
| `ask` | `string` | Subtype when `type` is `"ask"` |
| `reasoning` | `string` | Optional model reasoning |
| `partial` | `boolean` | Streaming flag |
