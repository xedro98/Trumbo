The Trembo API opens one endpoint to models from many providers. Model IDs use the `provider/model-name` format, the same convention popularized by [OpenRouter](https://openrouter.ai).

## Model ID Format

Every model is addressed by a string of the form:

```
provider/model-name
```

A few examples:
- `anthropic/claude-sonnet-4-6` - Claude Sonnet 4.6 from Anthropic
- `openai/gpt-4o` - GPT-4o from OpenAI
- `google/gemini-2.5-pro` - Gemini 2.5 Pro from Google

Pass that string as the `model` parameter in your [Chat Completions](/api/chat-completions) request:

```bash
curl -X POST http://0.0.0.0:0/api/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "minimax/minimax-m2.5",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

## Reasoning Models

Some models think through a problem before they answer. When you use one of these:

- Reasoning content streams through `delta.reasoning`
- Some providers also return encrypted reasoning blocks in `delta.reasoning_details`
- Reasoning tokens are metered separately from output tokens

Reasoning-capable families include most Claude, Gemini 2.5, and Grok 3 models. Confirm support for a specific model by checking its `supportsReasoning` capability in the model catalog.

## Choosing a Model

| If you need... | Consider |
|----------------|----------|
| Best coding performance | `anthropic/claude-sonnet-4-6` |
| Long document analysis | `google/gemini-2.5-pro` (1M context) |
| Fast, cheap responses | `deepseek/deepseek-chat` |
| Free experimentation | `minimax/minimax-m2.5` |
| Multi-modal (text + images) | `openai/gpt-4o` or `anthropic/claude-sonnet-4-6` |
| Complex reasoning | Any model with reasoning support |

For account and setup details, see the [Trembo Provider guide](/getting-started/trembo-provider).

## Image Support

Image-capable models accept base64-encoded image content inside the `messages` array:

```json
{
  "model": "anthropic/claude-sonnet-4-6",
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "What's in this image?"},
        {"type": "image_url", "image_url": {"url": "data:image/png;base64,..."}}
      ]
    }
  ]
}
```

Not every model accepts images. Check the model's `supportsImages` capability before sending image content.
