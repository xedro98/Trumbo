The Trumbo CLI ships a `plugin install` command that installs plugins from a variety of sources. You can also load plugins programmatically through session config.

## CLI Installation

Use the `trumbo plugin install` command to install plugins from file URLs, npm, git repositories, or local paths.

```sh
trumbo plugin install <source> [options]
```

The shorthand `trumbo plugin i` works as well.

### Install from a File URL

```sh
trumbo plugin install https://github.com/xedro98/trembo/blob/main/engine/examples/plugins/weather-metrics.ts
trumbo plugin install https://raw.githubusercontent.com/xedro98/trembo/main/engine/examples/plugins/weather-metrics.ts
```

Installs a single `.ts` or `.js` plugin file from an HTTPS URL. GitHub `blob` URLs are normalized to raw file downloads automatically.

### Install from npm

```sh
trumbo plugin install --npm @scope/plugin-name
```

Installs the package from the npm registry into the Trumbo plugin directory.

### Install from Git

```sh
trumbo plugin install --git https://github.com/owner/repo.git
trumbo plugin install --git github.com/owner/repo
```

Clones the repository (shallow, with blobless filter for speed) and installs its dependencies. The `.git` directory is excluded from the final install.

### Install from Local Source

```sh
trumbo plugin install ./path/to/plugin.ts     # Single file
trumbo plugin install /absolute/path/to/dir   # Directory with package.json
```

Supports both files and directories. Local installs copy the source, filter out `.git` and `node_modules`, and run `npm install` to resolve dependencies.

### Options

| Option | Description |
|--------|-------------|
| `--npm` | Treat the source as an npm package name |
| `--git` | Treat the source as a git repository URL |
| `--force` | Overwrite an existing plugin at the same path |
| `--json` | Output results as JSON (useful for scripting) |
| `--cwd` | Install to `<path>/.trumbo/plugins` and resolve relative local paths from `<path>` |

### Examples

```sh
# Install a local plugin directory
trumbo plugin install ./my-custom-plugin

# Install a single plugin file from GitHub
trumbo plugin install https://github.com/xedro98/trembo/blob/main/engine/examples/plugins/weather-metrics.ts

# Install an npm package as a plugin
trumbo plugin install --npm @trumbo/plugin-sql

# Install from a GitHub repo
trumbo plugin install --git https://github.com/my-org/my-plugin.git

# Force reinstall an existing plugin
trumbo plugin install --force --npm @scope/plugin-name

# Use shorthand
trumbo plugin i --git github.com/owner/repo
```

### Listing Installed Plugins

Installed plugins appear in the `plugins` section of your CLI config:

```sh
trumbo config
```

### Find All Your Plugins from your file system

Project-scoped plugins live in the `.trumbo/plugins/` folder at your workspace root. The CLI auto-discovers plugins from this folder when you run Trumbo in that project.

```text
your-project/
├── .trumbo/
│   └── plugins/
│       ├── my-plugin.ts
│       └── another-plugin/
└── ...
```

To add a single-file plugin to a project, install it with `--cwd`:

```sh
trumbo plugin install https://github.com/owner/repo/blob/main/plugins/my-plugin.ts --cwd .
```

Global plugins are discovered from `~/.trumbo/plugins/`, and Trumbo may also discover plugins from the system Plugins folder. Use `--cwd .` when you want plugins to stay with a specific project.

## Programmatic Installation

### Using `pluginPaths` (TrumboCore)

In SDK code, load plugins from file paths using the `pluginPaths` session config option:

```typescript
import { TrumboCore } from "@trumbo/sdk"

const trumbo = await TrumboCore.create({ clientName: "my-app" })

await trumbo.start({
  config: {
    systemPrompt: "Analyze this project",
    // ...model/runtime config
    pluginPaths: ["/absolute/path/to/plugin.ts"],
  },
})
```

Plugin files must export an `AgentPlugin` as the default export. `pluginPaths` also accepts a package directory -- the SDK reads `package.json` and follows the `trumbo.plugins` entries, so you can `npm install` once inside a package and iterate without re-running `trumbo plugin install` on every edit.

### Using `plugins` (Agent Runtime)

When using the `Agent` or `AgentRuntime` directly, pass plugin instances in the `plugins` array:

```typescript
import { Agent } from "@trumbo/sdk"
import { myPlugin } from "./my-plugin"

const agent = new Agent({
  providerId: "anthropic",
  modelId: "claude-sonnet-4-6",
  plugins: [myPlugin],
})
```

### Using `extensions` (TrumboCore)

With `TrumboCore`, use the `extensions` config array instead:

```typescript
import { TrumboCore } from "@trumbo/sdk"

const trumbo = await TrumboCore.create({ clientName: "my-app" })

await trumbo.start({
  config: {
    systemPrompt: "Analyze customer records",
    extensions: [myPlugin],
  },
})
```
