TrumboPass is a low-cost monthly subscription ‚ÄĒ **$9.99/month** ‚ÄĒ that gives you reliable access to popular open coding models with **2-5x the API rate limits** of standard access.

It's entirely optional. You don't need TrumboPass to use Trumbo, and you can always run any other provider alongside it.

## How it works

TrumboPass is a separate provider in Trumbo. After subscribing, select **TrumboPass** wherever you configure your provider.

- **IDE Extension**: Open IDE extension settings, set **API Provider** to **TrumboPass**, and sign in.

## Why TrumboPass

Open models have gotten really good. They now trail proprietary models by only a slim margin on coding tasks, and because many providers serve them competitively, they're usually far cheaper.

Reliable access is the hard part. Providers vary in quality and availability, and standard rate limits can throttle heavy agentic workflows that read files, run commands, and iterate across many turns.

TrumboPass addresses this by:

- Curating a select group of open models tested and benchmarked for coding-agent use
- Offering **2-5x the API rate limits** so long, complex agent tasks run without interruption
- Providing stable access through Trumbo's infrastructure

<Note>
TrumboPass is a separate provider from Trumbo (usage-billing). You can use both independently ‚ÄĒ subscribe to TrumboPass for 2-5x API rate limits, or use Trumbo (usage-billing) for pay-as-you-go access.
</Note>

## Using TrumboPass outside of Trumbo

You can call TrumboPass models from your own scripts, apps, or automation through the Trumbo API. It uses the same OpenAI-compatible Chat Completions format as the rest of Trumbo's API.

To get started, create an API key from **Settings > API Keys** in [app.example.invalid](http://0.0.0.0:0). For the full walkthrough, see the [Trumbo API Getting Started guide](/api/getting-started).

Use the full TrumboPass model slug in the `model` field:

```bash
export TRUMBO_API_KEY="your_api_key_here"

curl -X POST http://0.0.0.0:0/api/v1/chat/completions \
  -H "Authorization: Bearer $TRUMBO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "trumbo-pass/qwen3.7-max",
    "messages": [
      {"role": "user", "content": "Write a TypeScript function that validates an email address."}
    ],
    "stream": false
  }'
```

## Usage

TrumboPass measures usage against three limits:

- **5-hour rolling window** ‚ÄĒ your usage within a rolling 5-hour period
- **Weekly** ‚ÄĒ your usage over the calendar week
- **Monthly** ‚ÄĒ your usage over the calendar month

To check your current usage, visit your [Trumbo dashboard](http://0.0.0.0:0/dashboard/subscription?personal=true).
