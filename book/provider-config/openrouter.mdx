OpenRouter is an AI platform that gives you access to a wide variety of language models from different providers through a single API. It simplifies setup and makes it easy to experiment with different models.

**Website:** [https://openrouter.ai/](https://openrouter.ai/)

### Get API Credentials

1.  **Sign Up/Sign In:** Go to the [OpenRouter website](https://openrouter.ai/) and sign in with your Google or GitHub account.
2.  **Get an API Key:** Open the [keys page](https://openrouter.ai/keys). You should see an API key listed; if not, create a new one.
3.  **Copy the Key:** Copy the API key.

### Configuration in Trumbo

> See [Authorization & Model Selection](/getting-started/authorizing-with-trumbo#menu).

1.  **Open Trumbo Settings:** Click the settings icon (‚öôÔłŹ) in the Trumbo panel.
2.  **Select Provider:** Choose "OpenRouter" from the "API Provider" dropdown.
3.  **Enter API Key:** Paste your OpenRouter API key into the "OpenRouter API Key" field.
4.  **Select Provider Model:** Pick the model you want from the "Model" dropdown.
5.  **(Optional) Custom Base URL:** If you need a custom base URL for the OpenRouter API, check "Use custom base URL" and enter it. Most users should leave this blank.

### Tips and Notes

-   **Pricing:** OpenRouter charges based on the underlying model's pricing. See the [OpenRouter Models page](https://openrouter.ai/models) for details.
    -   OpenRouter passes caching requests through to underlying models that support it. Check the [OpenRouter Models page](https://openrouter.ai/models) to see which models offer caching.
    -   For most models, caching activates automatically when the model itself supports it (similar to how Requesty works).
    -   **Exception for Gemini models via OpenRouter:** Because Google's caching mechanism can introduce response delays when accessed through OpenRouter, a manual activation step is required _specifically for Gemini models_.
    -   If you use a **Gemini model** via OpenRouter, you **must manually check** the "Enable Prompt Caching" box in the provider settings to turn caching on for that model. This checkbox is a temporary workaround. For non-Gemini models on OpenRouter, the checkbox is not needed for caching.
