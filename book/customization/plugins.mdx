Plugins extend Trumbo with custom tools, lifecycle hooks, slash commands, and more. You can install them globally (available in all sessions) or per-project.

## Installing Plugins via CLI

The `trumbo plugin install` command installs plugins from four source types:

<Tabs>
  <Tab title="File URL">
    ```bash
    trumbo plugin install https://github.com/owner/repo/blob/main/plugins/my-plugin.ts
    trumbo plugin install https://raw.githubusercontent.com/owner/repo/main/plugins/my-plugin.ts
    ```

    File URLs install a single `.ts` or `.js` plugin file directly. GitHub `blob` and raw URLs are supported, and remote plugin file URLs must use `https://`.
  </Tab>
  <Tab title="Git Repository">
    ```bash
    trumbo plugin install https://github.com/owner/repo.git
    trumbo plugin install git@github.com:owner/repo.git
    ```

    The installer clones the repository, installs production dependencies, and registers the plugin entry files.

    To install a specific branch or tag, append `@ref`:

    ```bash
    trumbo plugin install https://github.com/owner/repo.git@v1.2.0
    trumbo plugin install https://github.com/owner/repo.git@main
    ```
  </Tab>
  <Tab title="npm Package">
    ```bash
    trumbo plugin install npm:@scope/my-plugin
    trumbo plugin install --npm my-plugin
    ```
  </Tab>
  <Tab title="Local Path">
    ```bash
    trumbo plugin install ./my-plugin
    trumbo plugin install ~/plugins/my-tool
    trumbo plugin install /absolute/path/to/plugin.ts
    ```

    Local installs copy the file or directory into the plugin store. Both single `.ts`/`.js` files and directories with a `package.json` are supported.
  </Tab>
</Tabs>

Additional flags:

| Flag | Description |
|------|-------------|
| `--force` | Replace an existing install for the same source |
| `--json` | Output the result as JSON (useful for scripting) |
| `--cwd <path>` | Install to `<path>/.trumbo/plugins` instead of the global directory |

After installation, confirm the plugin is loaded by running `trumbo config` and checking the plugin tab.

### Example: TypeScript Navigation Plugin

The [typescript-lsp-plugin](https://github.com/xedro98/trembo) is a good reference for how plugins work. It adds a `goto_definition` tool that uses the TypeScript Language Service API to resolve symbol definitions through imports, re-exports, and type aliases.

Install it with:

```bash
trumbo plugin install https://github.com/xedro98/trembo.git
```

Once installed, Trumbo can call `goto_definition` with a file path and line number to find where symbols are defined, which is much more precise than text search.

## Plugin Manifest Format

For a repository or npm package to be installable as a Trumbo plugin, its `package.json` should include a `trumbo` field that declares plugin entry points:

```json
{
  "name": "my-trumbo-plugin",
  "version": "1.0.0",
  "trumbo": {
    "plugins": [
      {
        "paths": ["./index.ts"],
        "capabilities": ["tools", "hooks"]
      }
    ]
  }
}
```

The `trumbo.plugins` array accepts:

| Format | Example |
|--------|---------|
| Object with `paths` array | `{ "paths": ["./src/plugin.ts"], "capabilities": ["tools"] }` |
| Plain string | `"./index.ts"` |

Each path should point to a `.ts` or `.js` file that exports an `AgentPlugin` (either as the default export or a named export).

If no `trumbo.plugins` field is present, the installer falls back to auto-discovery: it looks for standard entry points, then recursively scans for `.ts` and `.js` files (skipping `node_modules` and `.git`).

### Host-Provided Dependencies

Dependencies under the `@trumbo/` scope are provided by the host runtime. The installer automatically strips these from the plugin's dependency list before running `npm install`, so declare any `@trumbo/*` package your plugin imports as an optional peer dependency. The host currently provides `@trumbo/sdk`, `@trumbo/core`, `@trumbo/agents`, `@trumbo/llms`, and `@trumbo/shared`.

```json
{
  "peerDependencies": {
    "@trumbo/sdk": "*"
  },
  "peerDependenciesMeta": {
    "@trumbo/sdk": {
      "optional": true
    }
  }
}
```

## Plugin Directory Structure

Plugins are stored in the `plugins` directory at two levels:

```
~/.trumbo/
  plugins/                     # Global plugins
    _installed/                # Managed by `trumbo plugin install`
      npm/                     # npm-sourced plugins
      git/                     # git-sourced plugins
      remote/                  # file URL-sourced plugins
      local/                   # local-sourced plugins

.trumbo/                        # Project root
  plugins/                     # Project-scoped plugins
```

Global plugins (`~/.trumbo/plugins/`) are available across all sessions. Project plugins (`.trumbo/plugins/` in your repo) are available only when working in that project.
