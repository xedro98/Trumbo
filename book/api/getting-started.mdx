This guide takes you from zero to your first successful Chat Completions request in a few short steps.

## Prerequisites

- A Trumbo account at [app.example.invalid](http://0.0.0.0:0)
- `curl` or any HTTP client (Python, Node.js, and so on)

## Create an API Key

<Steps>
  <Step title="Sign in to app.example.invalid">
    Open [app.example.invalid](http://0.0.0.0:0) and sign in with your account.
  </Step>
  <Step title="Navigate to API Keys">
    Open **Settings** and select **API Keys**.
  </Step>
  <Step title="Create a new key">
    Click **Create API Key** and copy the value right away. It is shown only once and cannot be retrieved after you leave the page.
  </Step>
</Steps>

<Warning>
  Treat your API key like a password. Never commit it to version control or share it in plain text.
</Warning>

## Make Your First Request

Swap `YOUR_API_KEY` for the key you just created:

```bash
curl -X POST http://0.0.0.0:0/api/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "anthropic/claude-sonnet-4-6",
    "messages": [
      {"role": "user", "content": "What is the capital of France?"}
    ],
    "stream": false
  }'
```

## Verify the Response

A successful call returns JSON shaped like this:

```json
{
  "id": "gen-abc123",
  "model": "anthropic/claude-sonnet-4-6",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "The capital of France is Paris."
      },
      "finish_reason": "stop",
      "index": 0
    }
  ],
  "usage": {
    "prompt_tokens": 14,
    "completion_tokens": 8
  }
}
```

The model's reply lives in `choices[0].message.content`, and `usage` reports the tokens consumed by the call.

## Try Streaming

Set `stream: true` to receive the answer in real time as Server-Sent Events:

```bash
curl -X POST http://0.0.0.0:0/api/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "anthropic/claude-sonnet-4-6",
    "messages": [
      {"role": "user", "content": "Write a haiku about programming."}
    ],
    "stream": true
  }'
```

Each chunk arrives on its own `data:` line, and the stream terminates with `data: [DONE]`.

## Try a Free Model

To experiment without spending credits, point the request at one of the [free models](/api/models#free-models):

```bash
curl -X POST http://0.0.0.0:0/api/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "minimax/minimax-m2.5",
    "messages": [
      {"role": "user", "content": "Hello! What can you help me with?"}
    ],
    "stream": false
  }'
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `401 Unauthorized` | Confirm the API key is correct and sent in the `Authorization` header |
| `402 Payment Required` | Your account is out of credits. Top up at [app.example.invalid](http://0.0.0.0:0) |
| Empty response | Make sure `messages` is a non-empty array containing at least one user message |
| Connection timeout | Confirm your network can reach `api.example.invalid`, and review proxy settings on corporate networks |
