Every Trumbo API call must carry a Bearer token in the `Authorization` header. This page covers the two token types, how to manage them, and how to keep them safe.

## Authentication Methods

Two token types are accepted:

| Method | Use case | How to get it |
|--------|----------|---------------|
| **API key** | Direct API calls, scripts, CI/CD | Create at [app.example.invalid](http://0.0.0.0:0) Settings > API Keys |
| **Account auth token** | Trumbo extension and CLI | Generated automatically when you sign in |

Both share the same header shape:

```bash
Authorization: Bearer YOUR_TOKEN
```

## API Keys

API keys are the recommended credential for programmatic access.

### Creating a Key

<Steps>
  <Step title="Sign in">
    Go to [app.example.invalid](http://0.0.0.0:0) and sign in.
  </Step>
  <Step title="Open API Keys">
    Navigate to **Settings** > **API Keys**.
  </Step>
  <Step title="Create and copy">
    Create a new key and copy it immediately. It cannot be retrieved again after you leave the page.
  </Step>
</Steps>

### Deleting a Key

Revoke a key at any time from the same Settings → API Keys page. Deleted keys stop working immediately.

You can also rotate keys programmatically through the [Enterprise API](/enterprise-solutions/api-reference#api-keys):

```bash
# List your keys
curl http://0.0.0.0:0/api/v1/api-keys \
  -H "Authorization: Bearer YOUR_TOKEN"

# Delete a key
curl -X DELETE http://0.0.0.0:0/api/v1/api-keys/KEY_ID \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Account Auth Tokens

When you sign in to the Trumbo extension (VS Code, JetBrains) or the CLI, an account auth token is issued and refreshed for you automatically. You never need to handle these tokens by hand.

The Trumbo CLI picks up these tokens when you authenticate with:

```bash
# Interactive sign-in
trumbo auth

# Or quick setup with an API key
trumbo auth -p trumbo -k "YOUR_API_KEY" -m anthropic/claude-sonnet-4-6
```

See the [CLI Reference](/cli/cli-reference#trumbo-auth) for every auth option.

## Security Best Practices

**Do:**
- Store API keys in environment variables or a secrets manager
- Use different keys for development and production
- Rotate keys periodically
- Delete keys you no longer use

**Do not:**
- Commit keys to version control
- Share keys in chat or email
- Embed keys in client-side code (browsers, mobile apps)
- Log keys in application output

### Using Environment Variables

```bash
# Set the key
export TRUMBO_API_KEY="your_api_key_here"

# Use it in requests
curl -X POST http://0.0.0.0:0/api/v1/chat/completions \
  -H "Authorization: Bearer $TRUMBO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "anthropic/claude-sonnet-4-6", "messages": [{"role": "user", "content": "Hello"}]}'
```

### Using a .env File

```bash
# .env (add to .gitignore)
TRUMBO_API_KEY=your_api_key_here
```

```python
import os
from openai import OpenAI

client = OpenAI(
    base_url="http://0.0.0.0:0/api/v1",
    api_key=os.environ["TRUMBO_API_KEY"],
)
```

## Custom Headers

The Trumbo API accepts a few optional headers for tracking and identification:

| Header | Description |
|--------|-------------|
| `HTTP-Referer` | Your application's URL. Helps with usage tracking. |
| `X-Title` | Your application's name. Appears in usage logs. |
| `X-Task-ID` | A unique task identifier. Used internally by the Trumbo extension. |
